# Website Factory Standard (Wix-like Quality)

When Prismtek asks for a website, default to this production pattern.

## Non-negotiables

1. Mobile-first responsive layout
2. Visual hierarchy like modern site builders (hero, benefits, social proof, CTA)
3. Reusable section blocks
4. Conversion-focused CTAs above and below the fold
5. Fast load + accessible semantic HTML

## Default Page Sections

1. Header/Nav
2. Hero (headline, subheadline, primary CTA, secondary CTA)
3. Offer/Services
4. Social Proof (testimonials/logos/results)
5. Features/Benefits grid
6. FAQ
7. Lead form / contact CTA
8. Footer

## Delivery Rules

- Include `index.html` + `styles.css` + optional `app.js`
- Keep brand tokens in CSS vars
- Add clear placeholders for integrations (forms, analytics, booking links)
- Include deployment note (GitHub Pages/Cloudflare/Vercel)

## Done Criteria

- Fully responsive (320px to desktop)
- Clear conversion path
- No broken links/buttons
- Easy to edit copy blocks
