# 2026-03-06

## Omni mission + product direction

- Prismtek clarified canonical Omni mission: Omni should become a sovereign AI model + API stack intended to replace cost-based AI services while remaining no-cost to users in current phase.
- Coverage target explicitly includes parity expectations vs ChatGPT, Ollama, Grok, Nano Banana, and PixelLab.ai class capabilities.

## OmniAPI engineering progress

- Built and iterated parity benchmark tooling:
  - Added benchmark packs (`v1`, `v1.1`) and runner (`omni_parity_eval.py`).
  - Produced evidence reports and category score breakdowns.
- Implemented code-explain/path fixes and prompt shaping in Omni API server code; parity eventually reached full pass on latest run in-session.
- Updated Omni provider/routing layer to include `nanobanana2` provider plumbing (config, adapters, router chains, execution path, provider verification script), currently endpoint-dependent for readiness.

## Telegram bridge improvements (Omni)

- Verified Telegram bridge routing to Omni local backend (`local-ollama`, `omni-core:phase2`) and added reliable `/status` model/backend reporting from live probe.
- Improved responsiveness UX and reliability:
  - typing indicator
  - retry/timeout controls
- Fixed reply issues:
  - stripped prompt-echo behavior
  - disabled quoted reply threading by default to reduce perceived message mirroring
- Added `/image` handling in bridge:
  - routes to Omni image generate endpoint
  - polls image job status
  - uploads generated image to Telegram via `sendPhoto`
