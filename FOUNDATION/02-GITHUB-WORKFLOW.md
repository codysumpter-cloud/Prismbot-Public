# GitHub Workflow (Simple + Scalable)

## Branch naming
- `feat/<feature-name>`
- `fix/<bug-name>`
- `docs/<topic>`

## Issue template
- Problem
- Goal
- Scope
- Acceptance criteria
- Out of scope

## Commit style
- `feat: add leaderboard API route`
- `fix: correct collision logic for creature battle`
- `docs: add setup instructions`

## PR checklist
- [ ] Works locally
- [ ] README updated
- [ ] Screenshots/GIF if UI changed
- [ ] No secrets committed

## Project organization
- One repo per serious project
- Keep `README.md`, `ROADMAP.md`, `CHANGELOG.md`
- Use milestones: `MVP`, `Beta`, `Public Launch`
